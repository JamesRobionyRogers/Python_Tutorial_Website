Try & Except in Python

Easy - Used In Creating Robust Code

Robust Code

Being a good programmer doesn't just mean following good programming conventions, it also means considering user input and considering cases where invalid inputs are not accepted by our programming.

Take this line of code as an example, we want our user to enter an integer for some reason:

number = int(input("Please enter an integer: "))

If the user enters in the string "five" for example, or even "5.1", these inputs are both numbers but not valid inputs for our program. Python will throw an error into the shell as follows:

 >>> 'Please enter an integer:'  five
		         
 Traceback (most recent call last):
   File "<pyshell#0>", line 1, in <module>
     number = int(input("please: "))
 ValueError: invalid literal for int() with base 10: 'five'

This is not a good error for our user to see. This is where we can use try and except. try and except does allow for specific errors to be targeted but for the purposes of Level 1, any case where using the code in the try codeblock throws an error, we use the code in the except block to stop the program from throwing an exception, giving meaningful feedback to the user.

try:
	number = int(input("Please enter an integer: "))
except:
	print("You didn't enter a valid integer")

Now in the case where the user will enter in a value that is not an integer, the next line will print out.

>>>  "Please enter an integer:"  five
>>>  "You didn't enter a valid integer"

The only problem is that our program will end after the except. How can we write our program to force our user to enter a valid input?

If you guess by using a while loop you'd be right!

again = True

while again:
	try:
      number = int(input("Please enter an integer: "))
	  again = False
	except:
	  print("You didn't enter a valid integer")
