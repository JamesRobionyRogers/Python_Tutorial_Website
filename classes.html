Classes in Python

Medium - Hard - Basicaly a Blueprint for Creating an Object

The Basics

Creating a Class

Python is an object oriented programming language. This means that Python organises software design around data or objects, rather then functions and or logic.

An object is primarily a collection of data (in the form of variables) and methods (more commonly known as functions) which use that data.


Alright, now let's cut to the chase. Let's look at how you create a class.

We start off with the word class then follow it with the name of the class. In our case it'll be class Snake:

Something to note, the convention for naming classes is slightly different from functions and variables. We tend to use title case when naming classes. This is all according to the PEP8 standard.

class Snake:
	snake = "Python"

print(Snake)
>>>  "<class '__main__.Snake'>"

Creating an Object

Now that we have created our Snake class, we can assign it to a variable. This now makes this variable an object:

python = Snake()
print(python.snake)

>>>  "Python"

Using the __init__ Function

The above examples are the most basic examples of classes. You will virtually never use them in that form.

All classes have a built in function called __init__() which is always run when the class is initiated. For example when it is asigned to a variable.

We use the __init__() function to set any variables we may want to use within the class. Here is an example

class Snake:
   def __init__(self, species, colour):
       self.species = species
       self.colour = colour

# when you assign a class to a variable, it beomes known as an object
python = Snake('Python', 'Blue')
print(python.species, python.colour)

>>>  "Python Blue"

Methods Within an Object

Classes can also contain methods. Methods are functions that are specific to the class

Let's create a method called info() that will display the infomation given about the snake.

class Snake:
   def __init__(self, species, colour):
      self.species = species
      self.colour = colour

   def info(self):
      print(f"Species: {self.species}, Colour: {self.colour}")


python = Snake('Python', 'Blue')
python.info()
>>>  "Species: Python, Colour: Blue"

What Does self Do?

The self parameter is a reference back to the object of the class, and is used when you need to refer to a variable that belongs to the class.

self does not necessarily need to be named self. It can be whatever you want it to be, the only rule is that it must be the first parameter of any function of that class.

In this example as you can see I've changed self to object. Everything still works perfectly fine. 

class Snake:
   def __init__(object, species, colour):
     object.species = species
     object.colour = colour

   def info(object):
     print(f"Species: {object.species}, Colour: {object.colour}")


python = Snake('Python', 'Blue')
python.info()
>>>  "Species: Python, Colour: Blue"

Modifying The Properties in a Class

You can even modify properties of the object/class using a method.

In this example, as the method suggests, change_colour() will change the originally specified colour, to the newly specified colour.

class Snake:
   def __init__(self, species, colour):
       self.species = species
       self.colour = colour

   def info(self):
       print(f"Species: {self.species}, Colour: {self.colour}")

   def change_colour(self, new_colour):
       self.colour = new_colour


anaconda = Snake('Anaconda', 'Green')
anaconda.info()
>>>  "Species: Anaconda, Colour: Green"

anaconda.change_colour('Yellow')
anaconda.info()
>>>  "Species: Anaconda, Colour: Yellow"

Modifying the properties of an object is as easy as this:

anaconda = Snake('Anaconda', 'Green')

anaconda.species = "Python"
anaconda.colour = "Blue"

anaconda.info()
>>>  "Species: Python, Colour: Blue"
